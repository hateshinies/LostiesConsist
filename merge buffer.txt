Closes #1 

Предметная область: Сервис по поиску пропавших животных.

Сущности:
1. Adv. Объявление о пропаже/находке (pk - advId; fk - advCatgeory.category). Содержит название title и описание description. Является родителем для сущностей: Payment, AdvState, Response.
2. Response. Отклик на объявление (fk - adv.advId, responder.respId). Объявление получает Отклик. Отклик не может существовать без объявления. Он содержит сообщение с описанием description и внешние ключи к другим сущностям: Откликнувшийся(Responder) и Объявление(Adv). Сущность реализует связь многие ко многим между Adv и Responder.
3. Responder. Откликнувшийся (pk - respId) Человек, (который) откликнулся на объявление. Сущность хранит имя name, контактный телефон phone, электронную почту email. Является родительским объектом для Response, иными словами Responder делает Response.
4. AdvState. Статус объявления (pk - state, fk - adv.advId). Объявление имеет статус state со временем запуска start и остановки публикации end; у одного объявления могут быть разные статусы. Не может существовать без родительской сущности Adv. 
5. Payment. Информация об оплате объявления (pk - payId; fk - adv.advId). Объявление имеет информацию об оплате: sum,  method; оплат может быть несколько, напр. продление публикации, поэтому связь один ко многим. Родительской сущностью для Payment является Adv.
6. AdvCategory. Категория объявления (pk - category). Объявление имеет категорию. Содержит категории объявлений, напр. пропало животное или найдено.
7. Losty. Пропавшее животное (pk - lostyId, fk - owner.ownerId, advCategory.category). Животное, которое ищут. Дочерний объект владельца Owner. Содержит имя name и описание обстоятельств пропажи description, фотографию photo, категорию объявления category, идентификатор хозяина ownerId. Не может существовать без adv.Category, owner.ownerId. Дочерняя сущность Owner(-a). 
8. Foundy. Найденное животное (pk - foundyId, fk - finder.finderId, advCategory.category). Животное, которое нашлось. Дочерний объект нашедшего Finder. Содержит имя name и описание обстоятельств находки description, фотографию photo, категорию объявления category, идентификатор нашедшего finderId. Не может существовать без adv.advCategory, finder.finderId. Дочерняя сущность Finder(-a).
9. Owner. Владелец пропавшего животного (pk - ownerId). Если животное потерялось - его ищет хозяин. Хранит информацию о способах связи - name, phone, email. Родительская сущность пропавшего животного Losty.
10. Finder. Нашедший животное (pk - finderId). Если нашлось - нашедший сделал объявление. Сущность хранит информацию о способах связи - name, phone, email. Родительская сущность нашедшегося животного Foundy.

Сплошной линией с закрашенным кругом на конце обозначена связь один ко многим.
Пунктирной линией с пустым и закрашенным кругами обозначена связь один к одному.
